# Outfit Management Closure Acceptance Checklist

## 1. 推荐历史
- [ ] 在推荐页生成一组场景推荐后，打开“推荐历史”页面，能看到新记录
- [ ] 推荐历史中能看到场景、生成时间、推荐摘要
- [ ] 点击“标记采纳”后，状态更新为已采纳
- [ ] 填写“喜欢 / 一般 / 不喜欢”反馈后，刷新页面仍能看到反馈结果

## 2. 套装与推荐联动
- [ ] 在推荐页点击“加入套装库”后，套装保存成功
- [ ] 保存成功后，对应推荐历史的采纳状态被更新
- [ ] 套装库页面中能看到这条由推荐生成的套装

## 3. 穿搭记录
- [ ] 在推荐页点击“记录穿搭”后，穿搭记录创建成功
- [ ] 在衣橱页打开单件衣物详情，点击“记录穿搭”后，穿搭记录创建成功
- [ ] 打开“穿搭记录”页面，能看到最新记录
- [ ] 穿搭记录中能看到日期、场景、满意度和关联单品
- [ ] 删除一条穿搭记录后，列表正确更新

## 4. 偏好画像
- [ ] 打开“偏好画像”页面，能正常加载画像数据
- [ ] 点击“刷新画像”后，页面能显示最新更新时间
- [ ] 页面能展示偏好颜色、常用风格、高频场景、常用季节
- [ ] 当推荐反馈和穿搭记录增加后，画像摘要会发生合理变化

## 5. 衣橱分析
- [ ] 打开“衣橱分析”页面，能正常看到衣物总数、推荐次数、已采纳、采纳率
- [ ] 页面能展示类型分布、风格分布、颜色分布
- [ ] 页面能展示高频场景和穿搭趋势
- [ ] 当系统数据增加后，分析结果会同步更新

## 6. 基础验证
- [ ] 运行 `node --test server/tests/recommendations.helpers.test.js server/tests/outfitLogs.helpers.test.js server/tests/profileInsights.helpers.test.js server/tests/wardrobeAnalytics.helpers.test.js`
- [ ] 运行 `cd client && npm run lint`
- [ ] 数据库中存在以下表：
  - `recommendation_history`
  - `recommendation_feedback`
  - `outfit_logs`
  - `outfit_log_items`
  - `user_style_profile`

## 7. 答辩演示建议顺序
1. 在推荐页生成推荐
2. 展示推荐历史自动新增
3. 提交推荐反馈
4. 将推荐结果保存为套装
5. 将推荐结果记录为穿搭
6. 打开偏好画像页展示更新结果
7. 打开衣橱分析页展示闭环统计
