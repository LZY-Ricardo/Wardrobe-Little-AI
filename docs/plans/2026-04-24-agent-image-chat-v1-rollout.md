# Agent 图片直发聊天 V1 联调与迁移说明

## 1. 数据库迁移
本次 V1 依赖 `agent_messages.meta_json` 字段保存图片附件元数据。

在已有数据库上，至少执行以下 SQL：

1. 若尚未创建统一 Agent 表：
- `server/migrations/create_unified_agent_sessions.sql`

2. 若统一 Agent 表已存在：
- `server/migrations/alter_agent_messages_add_meta_json.sql`

## 2. 联调前检查
- `server/.env` 中 `SILICONFLOW_API_KEY` 已配置
- 视觉分析服务可用
- 统一 Agent 聊天接口可正常返回文本回复

## 3. 建议联调路径
### Case A：纯文本聊天
- 在 `AiChat` 中直接输入文本发送
- 预期：行为与改造前一致

### Case B：纯图发送
- 点击输入区 `+`
- 选择 `打开相册`
- 选中单张图片后，输入区上方出现待发送图片卡片
- 不输入文本，直接发送
- 预期：
  - 用户消息中出现图片卡片
  - Agent 返回基于图片的分析或引导回复

### Case C：图 + 文发送
- 选择图片
- 输入例如：`帮我看看这件衣服适合什么场景`
- 点击发送
- 预期：
  - 用户消息中同时保留图片与文本
  - Agent 回复同时参考图片和文本

### Case D：历史恢复
- 关闭当前会话，重新进入同一会话
- 预期：
  - 用户图片消息仍能显示缩略图
  - 多模态消息摘要不会展示原始 `[图片消息]` 或长分析文本

## 4. 当前 V1 已知限制
- 仅支持单图
- 不支持多图轮播
- 不支持图片消息重试按钮
- 不支持直接将聊天图片落入衣橱
- 图片分析失败时，仍以文本错误提示为主，未细化为更丰富的多模态错误卡片

## 5. 推荐下一步
- 增加图片消息重试入口
- 优化用户图片消息气泡的版式与占位
- 将图片消息摘要规则下沉为统一 preview helper，减少前后端分散逻辑
