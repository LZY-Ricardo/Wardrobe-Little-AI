# 偏好画像页与底部栏重设计规格

## 目标

把 `ProfileInsights` 从基础信息页提升为更接近成品的移动端页面，同时保持现有业务字段和交互不变。

## 对应实现

- 页面：`client/src/pages/ProfileInsights`
- 底部栏：`client/src/components/BottomNavigation`

## 页面结构

### 1. 页面容器

- 顶部留白控制在移动端常规内容页范围内。
- 内容区使用单列垂直流。
- 底部需要显式预留导航栏安全空间。

### 2. Hero 区

- 标题：`偏好画像`
- 副标题：`基于衣橱、穿搭记录与推荐反馈生成`
- 两个主操作：
  - `刷新画像`
  - `交给 Agent`
- 操作按钮放在首屏内，不折叠。

### 3. 摘要卡

- 卡头：`画像摘要`
- 主体：`data.summary`
- 辅助信息：
  - `更新时间`
  - `低风险操作免确认` 状态
- 状态按钮保留现有业务逻辑，只调整视觉表现。

### 4. 内容卡区

保留现有五个业务模块，不新增后端字段：

- `偏好颜色`
- `常用风格`
- `高频场景`
- `常用季节`
- `反馈偏好`

布局策略：

- 手机端双列卡片网格
- 标签为主，不引入重图表
- 空状态文案继续沿用当前业务语义

### 5. 趋势辅助卡

只做轻量视觉增强，不引入新接口字段。

- 标题：`反馈偏好趋势`
- 内容来源：现有反馈标签与摘要语义
- 作用是提升页面完成度，不作为精确数据展示

## 视觉口径

- 页面背景：浅灰到白的轻渐变
- 主卡片：白底、大圆角、轻阴影
- 主按钮：蓝色实心
- 次按钮：白底描边
- 标签：低饱和浅底色，按类别区分
- 标题层级：
  - 页面标题 24-28
  - 卡片标题 15-17
  - 正文 13-14
  - 辅助信息 11-12

## 底部栏规格

- 五项结构：
  - `衣序`
  - `衣藏`
  - `AI 入口`
  - `衣织`
  - `衣笺`
- 中间 AI 入口为浮起圆形按钮，强于普通 tab。
- 普通 tab 保持图标 + 文字的轻量导航样式。
- 保留现有隐藏/恢复导航栏机制。

## 实现约束

- 不改动接口契约。
- 不改动页面主交互语义。
- 不引入新状态管理。
- 允许调整布局、文案呈现、按钮结构和底部栏样式。
