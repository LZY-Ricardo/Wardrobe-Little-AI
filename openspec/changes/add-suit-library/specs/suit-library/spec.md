## ADDED Requirements

### Requirement: 套装库列表展示
系统 MUST 提供用户套装库列表页，展示当前登录用户已收藏/创建的套装集合。

#### Scenario: 用户打开套装库列表
- **WHEN** 用户已登录并进入套装库列表页
- **THEN** 系统返回该用户的套装列表（含套装封面、名称、场景、单品数量、创建时间）

#### Scenario: 套装库为空
- **WHEN** 用户套装库为空
- **THEN** 页面显示空态提示并引导去推荐或创建

### Requirement: 收藏推荐套装到套装库
系统 MUST 支持用户将推荐页生成的套装一键加入套装库，且对相同单品组合执行幂等收藏。

#### Scenario: 用户收藏一个推荐套装
- **WHEN** 用户在推荐页对某个套装点击收藏
- **THEN** 系统将该套装（单品组合）加入套装库并提示成功

#### Scenario: 用户重复收藏同一组合
- **WHEN** 用户收藏的套装单品组合在其套装库中已存在
- **THEN** 系统不重复创建记录并提示“已在套装库中”

### Requirement: 用户自定义创建套装
系统 MUST 支持用户从衣橱选择若干单品并创建一个自定义套装。

#### Scenario: 用户创建自定义套装成功
- **WHEN** 用户填写套装名称并选择至少 2 件单品提交
- **THEN** 系统创建套装记录并在套装库中可见

#### Scenario: 用户未选择足够单品
- **WHEN** 用户提交时选择的单品少于 2 件
- **THEN** 系统拒绝创建并返回可理解的错误提示

### Requirement: 删除套装
系统 MUST 支持用户从套装库删除套装。

#### Scenario: 用户删除套装
- **WHEN** 用户在套装库选择删除某套装并确认
- **THEN** 系统删除该套装及其关联单品关系，并在列表中移除
