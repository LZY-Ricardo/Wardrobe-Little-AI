## ADDED Requirements

### Requirement: 用户名密码认证规则必须由后端强制执行
系统 MUST 在注册与修改密码接口中由后端强制校验用户名和密码规则，而不能只依赖前端限制。

#### Scenario: 注册请求携带不符合长度规则的用户名
- **WHEN** 用户提交长度少于 6 位或多于 32 位的用户名进行注册
- **THEN** 服务端拒绝请求并返回明确的规则提示

#### Scenario: 修改密码请求携带不符合规则的新密码
- **WHEN** 用户提交长度少于 8 位或多于 64 位、缺少大写字母/小写字母/数字/特殊字符、包含空白字符，或与旧密码相同的新密码
- **THEN** 服务端拒绝请求并返回明确的规则提示

### Requirement: 新注册和新改密码必须满足强密码策略
系统 MUST 仅对新注册和修改密码操作强制执行强密码规则，而不能阻断历史弱密码用户继续登录。

#### Scenario: 老账号使用历史弱密码登录
- **WHEN** 已存在用户使用旧弱密码尝试登录
- **THEN** 只要用户名和密码匹配，系统仍允许登录，不因新密码策略直接拒绝

#### Scenario: 新注册使用弱密码
- **WHEN** 用户注册时提交不满足强密码规则的密码
- **THEN** 服务端拒绝注册并返回强密码规则提示

### Requirement: 登录注册输入必须做空白归一化
系统 MUST 在登录、注册和修改密码请求进入业务逻辑前对用户名和密码执行首尾空格裁剪，并拒绝裁剪后为空的值。

#### Scenario: 登录请求只包含空白字符
- **WHEN** 用户提交仅由空格组成的用户名或密码
- **THEN** 服务端拒绝请求并返回“用户名不能为空”或“密码不能为空”

### Requirement: JWT secret 必须由环境变量提供
系统 MUST 通过环境变量提供 JWT secret，且运行时不得依赖硬编码默认 secret。

#### Scenario: 服务端未配置 JWT_SECRET
- **WHEN** 认证模块启动并发现 `JWT_SECRET` 缺失或为空
- **THEN** 服务端立即报错，阻止继续使用硬编码默认值启动
