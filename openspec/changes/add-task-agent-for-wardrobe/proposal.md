# Change: 新增面向穿搭管理场景的任务型 Agent

## Why
当前项目中的 AI 能力主要分布在聊天、场景推荐和分析等若干入口中，整体仍表现为点状增强。用户要完成“查找适合某场景的衣物、生成推荐、保存为套装、记录今日穿搭”这类目标时，仍需在多个页面中手动跳转，AI 也无法将输出稳定沉淀为系统内可追踪的业务结果。

在已经具备业务闭环基础后，需要进一步引入一个面向穿搭管理场景的任务型 Agent，让 AI 能够理解用户自然语言目标、读取系统上下文、规划有限步骤、调用业务工具，并在统一确认机制下执行新增、修改、删除等写操作，从而真正贯穿系统功能。

## What Changes
- 新增任务型 Agent 独立页面，支持完整任务输入、任务理解、步骤展示、确认执行和结果返回。
- 新增全局悬浮入口与页面内快捷入口，使 Agent 能贯穿衣橱、套装、推荐、记录与画像等页面。
- 新增任务解析、上下文组装、有限步骤规划、工具调用与结果回写机制。
- 新增统一写操作确认机制，默认所有新增、修改、删除操作需要二次确认，并支持用户配置部分低风险操作免确认。
- 新增 Agent 任务历史与执行状态能力，用于追踪最近任务和执行结果。

## Impact
- Affected specs: `task-agent`
- Affected code:
  - Frontend: 新增 Agent 页面、全局悬浮入口、业务页快捷调用与确认交互
  - Backend: 新增任务解析与工具执行编排层、任务历史记录、统一确认控制
  - Data: 需要复用闭环方案中的推荐历史、穿搭记录、偏好画像，并新增 `agent_task_history`

## Non-Goals
- 不包含多 Agent 协同、自治长任务或复杂工作流引擎
- 不包含 RAG 平台、外部知识库基础设施或训练系统
- 不包含任何绕过安全门控的高风险自动写操作
