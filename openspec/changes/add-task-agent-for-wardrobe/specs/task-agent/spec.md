## ADDED Requirements
### Requirement: 系统 MUST 提供任务型 Agent 统一入口
系统 MUST 提供独立 Agent 页面、全局悬浮入口和业务页面快捷入口，使用户能够在不同页面上下文中发起 Agent 任务。

#### Scenario: 用户从独立页面发起任务
- **WHEN** 用户进入 Agent 页面并输入一个自然语言任务
- **THEN** 系统接收该任务并展示任务理解与执行结果

#### Scenario: 用户从业务页面快捷唤起 Agent
- **WHEN** 用户在衣橱页、推荐页或穿搭记录页使用快捷入口唤起 Agent
- **THEN** 系统能够携带当前页面上下文进入 Agent 任务流程

### Requirement: 系统 MUST 支持基于上下文的任务解析与有限步骤规划
系统 MUST 结合用户上下文、当前页面对象和长期偏好，对用户请求进行任务解析，并在需要时生成有限步骤的执行计划。

#### Scenario: Agent 解析查询类任务
- **WHEN** 用户询问适合通勤场景的现有衣物
- **THEN** 系统将任务识别为查询任务，并调用相关读工具返回结果

#### Scenario: Agent 解析组合任务
- **WHEN** 用户要求根据场景生成推荐并保存为套装
- **THEN** 系统将任务拆解为生成推荐、等待确认和保存套装等有限步骤

### Requirement: 系统 MUST 通过标准业务工具执行任务
系统 MUST 通过受控的后端业务工具完成 Agent 任务，而不是直接绕过业务层访问数据库。

#### Scenario: Agent 调用读工具
- **WHEN** Agent 需要查询用户的衣橱、套装或画像信息
- **THEN** 系统通过对应的标准读工具获取结构化结果

#### Scenario: Agent 调用写工具
- **WHEN** Agent 在用户确认后执行保存套装或创建穿搭记录
- **THEN** 系统通过对应的标准写工具完成数据写入

### Requirement: 系统 MUST 对所有写操作执行统一确认控制
系统 MUST 对新增、修改、删除等写操作执行统一确认控制，并允许用户为部分低风险操作配置免确认模式。

#### Scenario: 创建操作要求确认
- **WHEN** 用户要求 Agent 将推荐结果保存为套装
- **THEN** 系统先返回待执行摘要并等待用户确认后再执行写入

#### Scenario: 删除操作始终确认
- **WHEN** 用户要求 Agent 删除一条穿搭记录或删除套装
- **THEN** 系统必须进入确认流程且不允许绕过确认直接执行

### Requirement: 系统 MUST 提供 Agent 任务历史与执行状态追踪
系统 MUST 记录用户最近的 Agent 任务摘要、执行状态和关联业务对象，供前端展示与后续追踪。

#### Scenario: 用户查看最近任务
- **WHEN** 用户打开 Agent 页面或任务历史视图
- **THEN** 系统返回最近任务的摘要、状态和结果说明

#### Scenario: 写操作完成后写入任务历史
- **WHEN** Agent 成功执行一次创建、修改或删除任务
- **THEN** 系统记录该任务的类型、结果和关联对象信息
