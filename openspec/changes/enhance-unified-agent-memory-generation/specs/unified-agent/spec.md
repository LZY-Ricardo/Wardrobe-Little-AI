## ADDED Requirements
### Requirement: 系统 MUST 支持使用 LLM 生成统一 Agent 的会话标题
系统 MUST 优先使用 LLM 为统一 Agent 会话生成短标题，并在模型不可用或输出不合格时回退到规则标题。

#### Scenario: 会话首次形成明确主题
- **WHEN** 一个新会话产生了足够的主题信息
- **THEN** 系统为该会话生成一个简洁、主题化的标题

#### Scenario: 模型不可用时回退规则标题
- **WHEN** 标题生成模型不可用或输出不符合约束
- **THEN** 系统回退到规则标题生成逻辑，并保证会话标题仍可用

### Requirement: 系统 MUST 支持使用 LLM 生成结构化会话摘要
系统 MUST 优先使用 LLM 生成结构化会话摘要，并在模型不可用或输出不合格时回退到规则摘要，同时保持恢复契约不变。

#### Scenario: 会话消息超过 12 轮时生成结构化摘要
- **WHEN** 单个会话的消息超过 12 轮
- **THEN** 系统生成包含 `summary`、`key_facts`、`active_goals`、`pending_actions` 的结构化摘要

#### Scenario: 模型不可用时回退规则摘要
- **WHEN** 摘要生成模型不可用或返回无效结构
- **THEN** 系统回退到规则摘要，但仍返回完整结构化字段

### Requirement: 系统 MUST 在关键状态变化时刷新标题或摘要
系统 MUST 在会话主题稳定或任务状态发生关键变化时刷新会话标题或摘要，以保持恢复内容与当前会话状态一致。

#### Scenario: 新会话标题从默认值更新
- **WHEN** 一个会话标题仍为默认值且系统已识别出稳定主题
- **THEN** 系统更新该会话标题

#### Scenario: 写操作状态变化触发摘要刷新
- **WHEN** 会话中的写操作从待确认变为已执行、已取消或已过期
- **THEN** 系统刷新该会话的结构化摘要
