## ADDED Requirements
### Requirement: 统一 Agent MUST 支持单张图片直发聊天 V1
系统 MUST 允许用户在统一 Agent 聊天页中直接选择并发送单张图片，而不必强制跳转到独立上传页面。

#### Scenario: 用户在聊天页直接发送单张图片
- **WHEN** 用户在聊天输入区选择一张图片并点击发送
- **THEN** 系统在当前会话中接收该图片消息并继续生成 Agent 回复

#### Scenario: 用户发送图片与文本组合消息
- **WHEN** 用户同时选择一张图片并输入文本说明
- **THEN** 系统在同一条会话消息中处理图文输入，并返回结合图片与文本的回复

### Requirement: 系统 MUST 在聊天输入区展示待发送图片预览
系统 MUST 在图片真正发送前，于聊天输入区附近展示待发送图片缩略预览，以便用户确认当前发送内容。

#### Scenario: 用户选图后看到待发送预览
- **WHEN** 用户从相册中选中图片
- **THEN** 输入区展示该图片的待发送缩略图，而不是无反馈地停留在原状态

#### Scenario: 用户移除待发送图片
- **WHEN** 用户取消待发送图片
- **THEN** 系统移除该缩略预览，并恢复为普通文本输入状态

### Requirement: 系统 MUST 在统一会话中持久化图片消息与图文消息
系统 MUST 将图片消息或图文消息作为统一 Agent 会话中的正式消息持久化，并支持后续恢复与展示。

#### Scenario: 用户重新打开包含图片消息的历史会话
- **WHEN** 用户恢复一段包含图片消息的旧会话
- **THEN** 系统能够恢复该图片消息的可展示内容，而不是丢失或退化为不可读原始数据

#### Scenario: 图文消息进入会话列表摘要
- **WHEN** 某段会话的最后一条消息为图片消息或图文消息
- **THEN** 系统在会话列表中输出简洁摘要，而不是展示长 base64 或原始 Markdown 垃圾文本

### Requirement: 系统 MUST 在图片分析失败时保持聊天会话稳定
系统 MUST 在图片分析失败、图片消息发送失败或图文消息处理失败时，保留用户可见的失败消息状态，并给出明确反馈，而不能直接丢失消息或破坏当前会话。

#### Scenario: 图片分析失败但消息不应消失
- **WHEN** 用户发送图片后，服务端图片分析失败
- **THEN** 用户侧消息仍保留在会话中，并带有失败状态或明确提示

#### Scenario: 纯文本聊天不受图片能力影响
- **WHEN** 用户继续使用纯文本聊天
- **THEN** 系统行为与现有文本聊天保持一致，不因图文能力引入额外失败或协议变化

### Requirement: 系统 MUST 将图片分析结果注入统一 Agent 上下文
系统 MUST 允许统一 Agent 在图文会话中使用结构化图片理解结果作为上下文，以支持 Agent 在同一会话中继续解释、建议和回答。

#### Scenario: 用户上传衣物图片并询问搭配建议
- **WHEN** 用户发送一张衣物图片并询问适合的场景或搭配
- **THEN** 系统基于图片工具结果与文本问题生成针对性的回复

#### Scenario: 用户只发送图片不发送文字
- **WHEN** 用户仅发送图片而没有附加文本
- **THEN** 系统仍能基于图片工具结果返回基础理解或引导性回复

### Requirement: 统一 Agent MUST 将图片理解能力暴露为可调用工具
系统 MUST 将图片理解能力以显式工具的形式暴露给主对话模型，使主模型能够在带图会话中自主决定是否调用图片解析工具，而不是依赖固定的服务端前置摘要分流。

#### Scenario: 模型判断需要查看图片后调用工具
- **WHEN** 用户发送图片并提出依赖图片内容的问题
- **THEN** 主对话模型可以调用 `analyze_image` 工具，请求对当前消息附件进行结构化分析

#### Scenario: 模型判断无需调用工具
- **WHEN** 用户发送图片但当前回复不依赖图片细节，或已有足够上下文
- **THEN** 主对话模型可以选择不调用图片工具，并直接进入最终回复生成

### Requirement: 系统 MUST 将图片工具结果回注给主对话模型
系统 MUST 在图片工具调用完成后，将结构化结果作为同一轮对话的工具结果回注给主对话模型，再由主对话模型生成最终对用户可见的回复。

#### Scenario: 工具调用成功后继续回复
- **WHEN** `analyze_image` 工具成功返回结构化图片理解结果
- **THEN** 系统将该结果作为工具输出注入主对话模型上下文，并继续生成最终用户回复

#### Scenario: 工具结果不应直接替代最终回复
- **WHEN** 图片工具返回了分析 JSON 或摘要
- **THEN** 系统不应直接将工具原始输出作为最终 assistant 消息，而应由主对话模型组织最终回复

### Requirement: 带图会话 MUST 支持工具阶段事件与最终文本流
系统 MUST 在带图聊天中提供可感知的阶段流式反馈；若发生图片工具调用，前端应先收到工具阶段事件，再收到最终文本流式内容。

#### Scenario: 图片工具调用时先显示阶段事件
- **WHEN** 主对话模型决定调用图片工具
- **THEN** 系统向前端发送明确的工具阶段事件，例如“正在分析图片”，而不是长时间无反馈

#### Scenario: 工具完成后仍以文本流结束
- **WHEN** 图片工具调用完成并回注成功
- **THEN** 系统继续以 SSE 文本流方式输出最终 assistant 回复，而不是仅返回一次性整段结果

### Requirement: 带图消息 MUST 避免被图片摘要错误驱动到任务分支
系统 MUST 将带图消息在 V1 中优先视为工具增强聊天，而不是把图片解析摘要中的关键词直接用于任务分类；纯文本消息继续沿用现有任务 Agent 分支逻辑。

#### Scenario: 普通图文问答不应误入画像任务
- **WHEN** 用户发送图片并询问“这双鞋适合晴天穿吗”之类的普通图文问题
- **THEN** 系统应优先走工具增强聊天链路，而不是因为图片结果中包含“风格”等关键词而进入画像任务

#### Scenario: 带图业务诉求仍先走工具增强聊天
- **WHEN** 用户发送图片并表达推荐、画像或分析诉求
- **THEN** 系统在 V1 中仍优先走工具增强聊天链路，以保证阶段反馈与最终文本流式输出体验
