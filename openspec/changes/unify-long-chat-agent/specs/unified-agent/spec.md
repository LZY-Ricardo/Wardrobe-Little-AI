## ADDED Requirements
### Requirement: 系统 MUST 提供统一长对话式 Agent 会话模型
系统 MUST 将长对话助手与任务型 Agent 融合为统一会话模型，并支持在同一会话中完成连续对话与任务执行。

#### Scenario: 用户在统一 Agent 页面持续对话
- **WHEN** 用户在统一 Agent 页面中连续输入多轮消息
- **THEN** 系统在同一会话上下文中保留对话连续性并返回连续回复

#### Scenario: 用户在同一会话中从问答切换到任务执行
- **WHEN** 用户先进行普通问答，再发起穿搭推荐或保存套装等任务
- **THEN** 系统在同一会话中完成任务理解、工具调用与结果返回

### Requirement: 系统 MUST 支持会话持久化与历史恢复
系统 MUST 将 Agent 会话、消息和会话摘要持久化到数据库，并允许用户恢复之前的会话。

#### Scenario: 用户查看历史会话列表
- **WHEN** 用户进入统一 Agent 页面
- **THEN** 系统返回该用户的历史会话列表与最后活动时间

#### Scenario: 用户恢复旧会话
- **WHEN** 用户打开一段历史会话
- **THEN** 系统恢复该会话最近 12 轮完整消息，并结合会话摘要继续对话

#### Scenario: 恢复接口返回完整会话上下文
- **WHEN** 前端请求恢复某个历史会话
- **THEN** 系统至少返回会话元信息、最近 12 轮完整消息、会话摘要记忆和长期偏好摘要

### Requirement: 系统 MUST 使用“最近 12 轮完整消息 + 会话摘要 + 长期偏好摘要”的分层记忆结构
系统 MUST 仅保留最近 12 轮完整消息原样注入上下文，并将更早的历史压缩为会话摘要，同时注入长期偏好摘要。

#### Scenario: 会话消息超过 12 轮
- **WHEN** 单个会话的消息数量超过 12 轮
- **THEN** 系统仅保留最近 12 轮完整消息，并将更早内容汇总为会话摘要供后续恢复与推理使用

#### Scenario: 长期偏好参与会话决策
- **WHEN** 用户在统一 Agent 中请求推荐或任务执行
- **THEN** 系统能够将长期偏好摘要与当前会话上下文一起注入 Agent 推理流程

#### Scenario: 会话摘要记录覆盖边界
- **WHEN** 系统刷新一段会话的摘要
- **THEN** 系统记录该摘要已经覆盖到的最后一条消息边界，用于后续增量恢复与摘要更新

### Requirement: 系统 MUST 在统一会话中支持多轮澄清、工具调用与确认执行
系统 MUST 允许 Agent 在同一会话中先澄清缺失信息，再执行工具调用，并对写操作继续保留确认机制。

#### Scenario: 任务信息不足时进入澄清
- **WHEN** 用户发起一个信息不足的写操作任务
- **THEN** 系统先在当前会话中提出澄清问题，而不是直接失败或直接执行

#### Scenario: 写操作在统一会话中确认执行
- **WHEN** 用户在统一会话中要求保存套装或记录穿搭
- **THEN** 系统在同一会话中返回确认摘要，并在用户确认后完成写入
