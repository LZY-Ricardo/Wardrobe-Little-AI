## ADDED Requirements

### Requirement: 天气服务 MUST 支持按城市或坐标查询指定日期预报
后端天气服务 MUST 支持使用 `city` 或 `lat`/`lon` 查询指定日期的天气预报；当 `date` 缺失时 MUST 默认查询当前日期；当城市解析或天气拉取失败时 MUST 返回稳定的兜底天气数据，而不是直接让调用方崩溃。

#### Scenario: 使用城市查询指定日期天气
- **WHEN** 调用方向天气服务传入合法 `city=上海` 和 `date=2026-04-26`
- **THEN** 系统先将城市解析为坐标，再返回该日期对应的天气现象、温度范围、体感范围与数据源信息

#### Scenario: 日期缺失时默认查询当天
- **WHEN** 调用方只传入 `city=杭州`，未提供 `date`
- **THEN** 系统默认使用当前日期查询天气，并在返回结果中显式包含实际使用的日期

#### Scenario: 城市解析或天气源失败时兜底
- **WHEN** 城市无法解析或外部天气源请求失败
- **THEN** 系统返回 `source=fallback` 的稳定天气结构，且包含城市名与日期字段
